# СКФУ События — бот для управления мероприятиями в MAX

Бот и мини-приложение для организации университетских мероприятий. Позволяет публиковать афишу, регистрировать участников, выдавать электронные билеты с QR-кодами и сканировать их на входе. Возможна выгрузка списка посетителей мероприятия в формате xlsx.

Репозиторий мини-приложения: https://github.com/Qwerty000101/qwerty000101.github.io<br>

Сетевой адрес мини-приложения: https://qwerty000101.github.io/ (функциональность доступна только при входе через MAX)

Основные возможности бота:
- Просмотр афиши мероприятий с пагинацией и детальной информацией
- Регистрация на события и получение QR-билетов прямо в чат
- Управление билетами (просмотр, повторная отправка QR, отмена)
- Создание и редактирование мероприятий (для администраторов)
- Статистика по каждому событию (кто зарегистрировался, кто пришёл)
- Список участников с возможностью экспорта в XLSX
- Сканирование билетов через мини-приложение с помощью камеры или ввода UUID вручную
- Профиль пользователя с выбором роли (студент / администратор)
- Гибкие фильтры по категориям, институтам и поиск по названию

Роли пользователей:
- Студент — просмотр афиши, регистрация на мероприятия, управление своими билетами
- Администратор — создание и удаление мероприятий, просмотр статистики и списков участников, экспорт отчётов, сканирование билетов

Стек технологий:
- Node.js + Express
- SQLite (better-sqlite3)
- MAX Bot API (maxhub/max-bot-api)
- MAX-UI 
- XLSX для генерации отчётов
- QRCode для создания билетов

Как запустить бота:
- Установите Node.js версии 18 или выше
- Клонируйте репозиторий
- Перейдите в папку bot_events
- Выполните npm install
- Создайте файл .env и добавьте строку BOT_TOKEN=ваш_токен_бота
- Запустите бота командой npm start
- Бот будет доступен в мессенджере MAX, а REST API для мини-приложения — на порту 3000
- Если хостинг мини-приложения требует https, то необходимо создать https-соединение. В данном случае мини-приложение размещено на github pages, поэтому без https связи между ботом и мини-приложением не будет.
